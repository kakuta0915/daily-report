## 学んだこと・気づいたこと

- コンポーネント固有のメモリのことを state という。
- ローカル変数は、レンダー間で保持されない。ローカル変数の値が変更されても React は気づかないし、レンダーをトリガしない。
- コンポーネントを新しいデータで更新するには、レンダー間でデータ保持し、新しいデータでコンポーネントをレンダーするように React に伝える必要がある。
  useState はこの 2 つを提供する。
- フックは、React がレンダーされている間のみ利用可能な特別な関数である。
- useState を呼び出すことは、コンポーネントに何かを覚えさせるよう React に指示を出すということである。
- state セッタ関数は、state 変数を更新し、React にコンポーネントの再レンダーをトリガする
- state をセットすることで React に再レンダーを要求している。
  UI がイベントに応答するには state を更新する必要がある。
  セット関数が React に UI の再レンダーを要求する。
