## 学んだこと・気づいたこと

下記のコードの意味を理解した。

```ts
// 開発中は常に最新データを取得し、本番ではキャッシュを使います
export const fetchCache =
  process.env.NODE_ENV === "development" ? "force-no-store" : "default-cache";
```

このコードは、
**「今の実行環境が 開発環境 か 本番環境 かによって、キャッシュの設定を切り替える」** という処理をしています。

`process.env.NODE_ENV` とは、実行環境を表す環境変数 です。
主に次のような値になります。

- `development` → ローカル開発中
- `production` → 本番環境
- `test` → テスト環境

これは日本語にすると、次の意味になります。

もし `NODE_ENV` が `'development'` なら `'force-no-store'` を使う。
そうでなければ `'default-cache'` を使う。

#### 開発環境の場合

- 毎回最新のデータを取得したい
- キャッシュがあると変更が反映されず混乱する

そのため、キャッシュを使わない設定 にしています。

#### 本番環境の場合

- パフォーマンスを良くしたい
- 毎回 API を叩くと遅くなる

そのため、キャッシュを有効にする設定 にしています。

まとめると、
「**開発中は毎回最新データ、本番では速さ優先**」という設定です。
