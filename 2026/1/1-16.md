## 学んだこと・気づいたこと

- generateStaticPrams は、「動的ルートを静的生成するための関数」である。
- 下記のコードの意味
  ```tsx
  export async function generateStaticParams(): Promise<{ slug: string }[]> {
    const slugs = await getAllSlugs();
    return slugs.map((item: { slug: string }) => ({ slug: item.slug }));
  }
  ```
  この関数は、「どの slug のページを静的生成するか」を Next.js に教えるためのものである。
  - この関数自体は引数を受け取らない。
  - slug を「使う側」ではなく「列挙する側」である。
    `item`1 つ 1 つは、`{ slug: string }` という形のオブジェクト。
  ```tsx
  ({ slug: item.slug });
  ```
  これは、
  1. `item.slug` の値を取り出し
  2. `{ slug: ○○ }` という 新しいオブジェクト を作って返す
     という内容の処理である。
- キャッシュとは、一度取得した結果を保存しておき、次回以降はそれを再利用する仕組みである。
- フェッチとは、簡単に言うと「外部にあるデータを取りに行く仕組み」である。ブラウザやサーバーから、別のサーバーに対して「この情報をください」とリクエストを送り、その結果を受け取る一連の流れを指す。
