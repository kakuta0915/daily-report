## 学んだこと・気づいたこと

#### 下記コードでは、分割代入をしているか？

```tsx
export async function generateMetadata(
  { params }: Props
): Promise<Metadata> {
  const { slug } = params
```

この `slug` は分割代入されています。
そして、`params` というオブジェクトの中から取り出されています。

1. **関数の引数での分割代入**

   ```tsx
   export async function generateMetadata({ params }: Props);
   ```

   `params` は Next.js が渡してくる値
   中身は `{ slug: string }`

2. **関数内での分割代入**

   ```tsx
   const { slug } = params;
   ```

   意味は、

   - `params` オブジェクトの中から
   - `slug` プロパティを取り出して
   - `slug` という変数に代入する

   という処理です。

#### `notFound()`関数の意味

この `notFound` は Next.js が用意している関数 です。
\*\*意味は一言で言うと、「このページは存在しない」と Next.js に伝えるための関数です。

notFound() を呼ぶとどうなるか\*\*
`notFound()` を実行すると、Next.js は次のように動きます。

1. 現在のページの描画を 即座に中断 します
2. HTTP ステータスコード 404 を返します
3. `app/not-found.tsx` があれば そのコンポーネントを表示 します
4. なければ Next.js のデフォルト 404 ページ を表示します

#### slug の正体

`slug` は Next.js のルーティング機構が自動で渡してくる値 です。
